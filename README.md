# СоСед — веб-сервис для поиска соседей по комнате или квартире

Дипломный проект. Полноценное веб-приложение с серверной частью:
регистрация пользователей и администраторов, вход по паролю с отправкой
на почту, умный подбор соседей по совместимости и управление анкетами
через админ-панель.

---

## Возможности

- **Регистрация и вход** — пользователь регистрируется по имени и почте,
  пароль генерируется автоматически и **приходит на электронную почту**.
- **Два типа аккаунтов** — обычный пользователь и администратор
  (регистрация админа защищена секретным кодом).
- **Админ-панель** — администратор видит все анкеты и всех пользователей,
  может удалять анкеты и удалять пользователей.
- **Каталог анкет** с умным подбором по совместимости (образ жизни, привычки).
- **Размещение анкеты** — доступно только авторизованным пользователям.
- **Безопасность** — пароли хранятся в виде хеша (bcrypt), вход по токену (JWT).

## Что нужно установить

**Node.js** (версия 18 или новее). Проверить, установлен ли он:

```
node -v
```

Если выводит версию (например `v20.11.0`) — всё готово.
Если команда не распознана — скачайте и установите Node.js с сайта
**https://nodejs.org** (кнопка LTS), затем перезапустите терминал.

## Быстрый запуск (на своём компьютере)

1. Распакуйте папку `sosed` и откройте её в VS Code (**File → Open Folder**).

2. Откройте встроенный терминал: меню **Terminal → New Terminal**
   (или **Ctrl+`**).

3. Установите зависимости (один раз):

   ```
   npm install
   ```

4. Запустите сервер:

   ```
   npm start
   ```

5. В терминале появится строка:
   `🏠 СоСед запущен! Локально: http://localhost:3000`

   Откройте в браузере **http://localhost:3000** — сайт работает.

> Чтобы остановить сервер — нажмите **Ctrl+C** в терминале.

## Как зарегистрироваться

На сайте нажмите **«Регистрация»** в правом верхнем углу.

- **Обычный пользователь** — введите имя и почту, нажмите «Зарегистрироваться».
- **Администратор** — выберите тип «Администратор» и введите код.
  Код по умолчанию: **`admin2026`** (его можно сменить в файле `.env`).

После регистрации пароль придёт на почту. **Если почта не настроена**
(см. следующий раздел), пароль покажется прямо на экране и в терминале —
так можно сразу всё протестировать без настройки почты.

Затем нажмите **«Вход»**, введите почту и полученный пароль.

## Настройка отправки пароля на почту (необязательно)

По умолчанию почта не настроена, и пароль показывается на экране.
Чтобы пароли действительно приходили на e-mail:

1. В папке проекта найдите файл `.env.example`, скопируйте его и
   переименуйте копию в `.env`.

2. Откройте `.env` и заполните данные почты. Пример для Gmail:

   ```
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=ваша_почта@gmail.com
   SMTP_PASS=пароль_приложения
   SMTP_FROM=ваша_почта@gmail.com
   ```

3. Для Gmail нужен **«пароль приложения»** (App Password), а не обычный
   пароль от аккаунта. Включите двухэтапную аутентификацию в Google,
   затем создайте пароль приложения здесь:
   **https://myaccount.google.com/apppasswords**
   Вставьте полученные 16 символов в `SMTP_PASS`.

4. Заодно поменяйте `JWT_SECRET` и `ADMIN_CODE` на свои значения.

5. Перезапустите сервер (**Ctrl+C**, затем снова `npm start`).
   В логе должно появиться `📧 Почта настроена`.

## Как открыть сайт с другого компьютера

Есть три способа — от самого простого к «настоящему хостингу».

### Способ 1 — по локальной сети (самый простой)

Подходит, если другой компьютер/телефон в той же Wi-Fi сети.

1. Запустите сервер (`npm start`).
2. Узнайте локальный IP-адрес вашего компьютера. В терминале Windows:

   ```
   ipconfig
   ```

   Найдите строку **IPv4-адрес**, например `192.168.1.42`.
3. На другом устройстве в той же сети откройте в браузере:
   **http://192.168.1.42:3000** (подставьте свой адрес).

> Если не открывается — Windows может блокировать порт. При первом запуске
> разрешите Node.js доступ в брандмауэре, либо временно отключите его для
> частной сети. Это работает только пока ваш компьютер включён и сервер запущен.

### Способ 2 — бесплатный хостинг Render.com (доступ из любой точки)

Сайт будет доступен по постоянной ссылке с любого устройства.

1. Загрузите проект на **GitHub**:
   - Создайте репозиторий на https://github.com
   - Через VS Code (вкладка **Source Control**) или командами Git
     отправьте папку `sosed` в репозиторий.
2. Зайдите на **https://render.com**, зарегистрируйтесь (можно через GitHub).
3. Нажмите **New → Web Service**, подключите свой репозиторий.
4. Настройки:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** Free
5. В разделе **Environment** добавьте переменные (как в `.env`):
   `JWT_SECRET`, `ADMIN_CODE`, и при желании `SMTP_*`.
6. Нажмите **Create Web Service**. Через пару минут Render выдаст ссылку вида
   `https://sosed.onrender.com` — её можно открыть откуда угодно.

> Важно: на бесплатном тарифе файлы с данными (`data/`) сбрасываются при
> перезапуске сервиса, поэтому новые анкеты и пользователи могут пропадать.
> Для постоянного хранения данных в дипломе можно подключить полноценную
> базу данных (например, PostgreSQL) — но для демонстрации работы и защиты
> текущего варианта достаточно.

### Способ 3 — быстрая публичная ссылка через ngrok (на время)

Даёт временный публичный адрес к вашему локальному серверу.

1. Скачайте **ngrok** с https://ngrok.com и зарегистрируйтесь.
2. Запустите ваш сервер (`npm start`).
3. В отдельном терминале выполните:

   ```
   ngrok http 3000
   ```

4. ngrok покажет ссылку вида `https://xxxx.ngrok-free.app` — её можно
   открыть с любого устройства, пока ваш компьютер и ngrok работают.

## Технологии

- **Бэкенд:** Node.js, Express
- **Авторизация:** JWT (jsonwebtoken), хеширование паролей bcryptjs
- **Почта:** Nodemailer
- **Хранение данных:** JSON-файлы (без установки СУБД)
- **Фронтенд:** HTML5, CSS3, JavaScript (без фреймворков)

## Тестовые сценарии для защиты

1. Зарегистрировать обычного пользователя → войти → разместить анкету.
2. Зарегистрировать администратора (код `admin2026`) → войти →
   открыть «Админ-панель» → удалить анкету и пользователя.
3. Проверить, что обычный пользователь не видит кнопок удаления.
4. Воспользоваться фильтрами и убедиться, что процент совпадения меняется.
